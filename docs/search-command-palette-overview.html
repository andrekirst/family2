#208 Universal Search & Command Palette

Complete implementation overview — how data is collected, transferred, and displayed

1 Architecture Overview

The command palette (Ctrl+K) combines three data sources into a unified interface: client-side NLP parsing, server-side GraphQL search, and static default items. Each source contributes items with a specific type that determines grouping, icons, and click behavior.

3 Data Transfer — What Gets Sent

3.1 GraphQL Request

query UniversalSearch($input: UniversalSearchRequestInput!) {
  search {
    universal(input: $input) {
      results { title, description, module, icon, route }
      commands { label, description, keywords, route,
                 requiredPermissions, icon, group }
    }
  }
}

// Variables:
{
  "input": {
    "query": "tomorrow event",   // User's search text
    "modules": null,             // Optional module filter
    "limit": 10,                 // Per-provider cap
    "locale": "en"               // From LOCALE_ID
  }
}

7 Default Items — Daily Rotation

Hint Rotation Algorithm

Two NLP hints are selected daily from a pool of 10 per locale, using deterministic epoch-day math. All users see the same hints on the same day. The pool rotates every 10 days.

11 Internationalization (i18n)

Frontend i18n

  • LOCALE_ID injection: selects EN or DE hint pool + NLP rules
  • NLP rules: separate en.rules.ts / de.rules.ts
  • Date parsing: "tomorrow" vs "morgen", day names in both languages
  • Time parsing: "3 PM" vs "15 Uhr"
  • Hint pool: 10 EN phrases, 10 DE phrases

Backend i18n

  • CommandDescriptor: optional LabelDe + DescriptionDe fields
  • Keywords: mixed-language arrays (e.g., ["invite", "einladen"])
  • Locale parameter: passed through GraphQL, used for label resolution
  • QueryHandler: ResolveGermanLabels() swaps labels when locale=de

12 Error Resilience

LayerError HandlingImpact
Frontend NLP Parser try/catch around parse() NLP fails silently; GraphQL search continues
Frontend SearchService Returns empty result on GraphQL error Palette shows "Search failed" error message
Backend Per-provider try/catch per SearchAsync() call One provider fails; other 7 continue normally
GraphQL Auth [Authorize] on QueryType Unauthenticated requests return GraphQL error
